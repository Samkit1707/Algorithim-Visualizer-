The 7 Layers of the TCP/IP Model

  • Layer 7: Application Layer

    Provides services directly usable by applications like web browsing (HTTP), email (SMTP), file transfer (FTP), and video streaming.

  • Layer 6: Presentation Layer

    Deals with data formatting and encryption/decryption. Ensures data is understood by both applications regardless of underlying systems.

  • Layer 5: Session Layer

    Establishes, manages, and terminates sessions between applications. Examples include login sessions or file transfer connections.

  • Layer 4: Transport Layer

    Provides reliable data transfer between applications. TCP (Transmission Control Protocol) ensures data arrives in order and without errors. UDP (User Datagram Protocol) prioritizes speed over reliability for applications like real-time video.

  • Layer 3: Network Layer

    Handles logical addressing (IP addresses) and routing packets across networks. It determines the best path for data to reach its destination.

  • Layer 2: Data Link Layer

    Controls how data is transmitted over physical media (cables, Wi-Fi) and handles error detection using techniques like checksums. It manages physical addressing (MAC addresses) of devices on the network.

  • Layer 1: Physical Layer

    Defines the physical transmission of data bits over a communication channel. It specifies the electrical or optical signals used to represent data on the medium.
